# Phase 6.1: Country in Alerts - Research

**Researched:** 2026-03-12
**Domain:** API field extraction (SportyBet + BetPawa)
**Confidence:** HIGH

<research_summary>
## Summary

Verified API response structures for both SportyBet and BetPawa to identify country/region fields. Both APIs provide country data at the tournament/competition level.

**SportyBet:** Country is in `categoryName` field at tournament level (e.g., "Turkiye")
**BetPawa:** Country is in `region.name` field at event level (e.g., "Turkey", "Indonesia")

Both platforms use different naming conventions (SportyBet: "Turkiye", BetPawa: "Turkey") but this is acceptable for display purposes.

**Primary recommendation:** Add `country_name` field to `LiveEvent` model, extract from `categoryName` (SportyBet) and `region.name` (BetPawa), update Slack message format.
</research_summary>

<api_structures>
## API Response Structures

### SportyBet Tournament Object
```json
{
  "id": "sr:tournament:98",
  "name": "1. Lig",
  "categoryName": "Turkiye",
  "categoryId": "sr:category:46",
  "events": [...]
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Tournament ID (sr:tournament:X) |
| `name` | string | League/competition name |
| `categoryName` | string | **Country name** |
| `categoryId` | string | Category ID (sr:category:X) |
| `events` | array | List of events in tournament |

### BetPawa Event Object
```json
{
  "id": "33509908",
  "name": "Psim Yogyakarta - Persijap Jepara",
  "region": {
    "id": "168",
    "name": "Indonesia"
  },
  "competition": {
    "id": "12352",
    "name": "Liga 1"
  },
  ...
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `region.id` | string | Region/country ID |
| `region.name` | string | **Country name** |
| `competition.id` | string | Competition/league ID |
| `competition.name` | string | League name |
</api_structures>

<implementation_approach>
## Implementation Approach

### 1. Model Change (`clients/models.py`)
Add optional `country_name` field to `LiveEvent`:
```python
class LiveEvent(BaseModel):
    ...
    country_name: str | None = None  # Country/region name for display
```

### 2. SportyBet Client (`clients/sportybet.py`)
Extract `categoryName` from tournament and pass to event:
```python
def _parse_single_event(
    self, event_data: dict, competition_id: str, competition_name: str, country_name: str
) -> LiveEvent | None:
    ...
    return LiveEvent(
        ...
        country_name=country_name,
    )
```

### 3. BetPawa Client (`clients/betpawa.py`)
Extract `region.name` from event:
```python
def _parse_single_event(self, event_data: dict) -> LiveEvent | None:
    ...
    region = event_data.get("region", {})
    country_name = region.get("name", "")
    ...
    return LiveEvent(
        ...
        country_name=country_name if country_name else None,
    )
```

### 4. Slack Notifier (`clients/slack.py`)
Update message format:
```python
def _format_message(self, event: LiveEvent) -> str:
    ...
    # Build competition line with optional country
    if event.country_name:
        competition_line = f"{event.country_name} - {event.competition_name}"
    else:
        competition_line = event.competition_name

    return f"🔴 *Missing on BetPawa*\n{match_line}\n{competition_line} | Score: {score}"
```

<edge_cases>
## Edge Cases

1. **Missing country data:** Some events may not have country populated. Use optional field with fallback to display competition name only.

2. **Country name variations:** Same country may have different names across platforms (e.g., "Turkiye" vs "Turkey"). Acceptable for display purposes - no normalization needed.

3. **International competitions:** Events like "Champions League" may have generic region or no region. Display competition name only if country is empty/generic.
</edge_cases>
