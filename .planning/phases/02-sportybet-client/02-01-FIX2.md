  <action>
The SportyBet API response structure uses different field names than currently implemented:

**Current (wrong):**
- `gameInfo.liveScore` - field does not exist
- `gameInfo.minute` - field does not exist

**Correct:**
- `setScore` - directly on event object (format: "1:0" or "1-0")
- `playedSeconds` - directly on event object (format: "MM:SS" like "09:05")

Update `_parse_single_event` method:

1. Replace `game_info = event_data.get("gameInfo", {})` with direct field access
2. Get score from `event_data.get("setScore", "")` instead of `game_info.get("liveScore", "")`
3. Get minute from `event_data.get("playedSeconds", "")` instead of `game_info.get("minute")`
4. Parse minute by extracting MM from "MM:SS" format (split on ":" and take first part)
5. Handle cases where playedSeconds might be empty or invalid

Example API data:
```json
{
  "setScore": "0:0",
  "playedSeconds": "09:05"
}
```
