# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-12
**Phases:** 1-6.5
**Total Plans:** 14

## Overview

Build a live event coverage comparison tool that monitors SportyBet and BetPawa for live football matches, alerts via Slack when events are missing from BetPawa. Starting with foundation and API clients, progressing through matching engine and alerts, finishing with monitoring loop and refinements.

## Milestone Summary

**Decimal Phases:**
- Phase 6.1: Country in Alerts (inserted for better alert context)
- Phase 6.2: SRL Filter & Provider Info (inserted to reduce noise)
- Phase 6.3: In-Play Filter (inserted to reduce false positives)
- Phase 6.4: Pre-match Cache (inserted to eliminate false positives)
- Phase 6.5: Human-Readable Logging (inserted for operations)

**Key Decisions:**
- httpx over aiohttp for modern async HTTP with better typing
- Pydantic v2 with PrivateAttr for provider ID override pattern
- Provider ID matching (exact) over fuzzy team name matching
- In-memory deduplication (no TTL - events expire naturally when no longer live)
- 5-minute pre-match cache refresh for balance of freshness vs API load

**Issues Resolved:**
- Pydantic underscore field restriction (used PrivateAttr)
- ruff linting (StrEnum, datetime.UTC)
- Tournament filtering in SportyBet client
- Widget parsing in BetPawa client

**Issues Deferred:**
- Docker deployment (Phase 7 removed from v1.0 scope)

**Technical Debt Incurred:**
- None significant - clean implementation
